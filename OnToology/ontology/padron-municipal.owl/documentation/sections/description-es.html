Vocabulario para la representación de los datos de padrón de habitantes: Descripción volver a índice

La Figura 1 muestra las clases y propiedades del Vocabulario de padrón de habitantes.
Como se mencionó previamente, el objetivo de este vocabulario es representar los datos que provienen del fichero de relación de habitantes que se emplea para el intercambio de datos de la población entre los Ayuntamientos y el Instituto Nacional de Estadística. Para ello, como se puede observar en la figura, se han definido varias clases y propiedades que permiten describir estos datos reutilizando principalmente la clase [schema:Person](http://schema.org/Person) y sus propiedades asociadas para representar el habitante. Adicionalmente, en este vocabulario se han utilizado varios esquemas de conceptos SKOS para representar algunos de los datos. En la Figura 1 se pueden observar varias relaciones con conceptos SKOS, sin embargo los valores concretos no se han especificado pues son diferentes según la taxonomía que representan. Por ejemplo, la propiedad [espad:tipoVariacion](#tipoVariacion) permite representar el tipo de variación del habitante que se registra en el Ayuntamiento, de acuerdo con un esquema de conceptos SKOS que identifica si se trata de un alta, baja, o modificación, y las causas asociadas a cada una de estas variaciones.

3.1 Ejemplos de uso volver a índice

Con la finalidad de facilitar la comprensión del uso de algunas clases y propiedades de este vocabulario se proporcionarán a continuación algunos ejemplos. En primer lugar para la representación de una variación se crea una instancia de la clase [espad:VariacionHabitante](#VariacionHabitante) y se enlaza, mediante la propiedad [espad:registraHabitante](#registraHabitante), con el habitante para el que se registra dicha variación . A continuación, se especifica la naturaleza de esta variación mediante la propiedad [espad:tipoVariacion](#tipoVariacion) que enlaza la variación y su causa, conforme un esquema de conceptos de tipos de variación, que para el caso de este ejemplo es un alta por nacimiento (ANA). Finalmente, se representa la fecha en la que esta variación entrará en vigencia, para ello se usa la propiedad [espad:fechaVariacion](#fechaVariacion).

A continuación, se representa el habitante asociado con la variación de Alta por Nacimiento definida previamente. En primer lugar se crea una instancia de la clase [espad:Habitante](#Habitante) y se define como activo, para ello se emplea la propiedad [espad:activo](#activo) que es un booleano que permite especificar si los datos del habitante están vigentes o no dentro del padrón. Luego se describen otros datos del habitante tales como su fecha de nacimiento ([schema:birthDay](http://schema.org/birthDay)), sexo ([schema:gender](http://schema.org/gender)), o el número de identificación que le asigna el Ayuntamiento ([espad:numeroIdentificacionAyuntamiento](#numeroIdentificacionAyuntamiento)). Adicionalmente, se definen la provincia y el municipio en el que ha nacido el habitante mediante las propiedades [espad:provinciaNacimiento](#provinciaNacimiento) y [espad:municipioNacimiento](#municipioNacimiento) respectivamente. Así también, se representan los detalles de donde reside el habitante que para el caso de nuestro ejemplo vienen dadas por la dirección, provincia, municipio, y distrito, descritas con las propiedades [schema:address](http://schema.org/address), [espad:provinciaResidencia](#provinciaResidencia), [espad:municipioResidencia](#municipioResidencia) y [espad:distritoResidencia](#distritoResidencia) respectivamente. Como se puede observar en la Figura 1 del diagrama del vocabulario, todas los detalles de la dirección postal y de las unidades administrativas de residencia del habitante se representan mediante las clases y propiedades que se reutiliza de la representación de [direccion-postal](http://vocab.linkeddata.es/datosabiertos/def/commons/direccion-postal#) y [territorio](http://vocab.linkeddata.es/datosabiertos/def/sector-publico/territorio#). Por lo tanto, los detalles de respresentación de todos estos conceptos se pueden revisar en detalle en la documentación disponible para cada uno de los vocabularios reutilizados. Finalmente, se especifica el nivel de estudios, el tipo de documento de identidad, y el tipo de local donde vive. Cada uno de estos datos se representan con sus respectivas propiedades ([espad:nivelEstudios](#nivelEstudios), [espad:tipoDocumentoIdentidad](#tipoDocumentoIdentidad), [espad:tipoLocalResidencia](#tipoLocalResidencia)) que enlazan al habitante con una lista de conceptos en el que se especifica cada uno de estos tipos. Por ejemplo, el nivel de estudios de nuestro ejemplo corresponde al código 00 que significa "No aplicable por ser menor de 16 años". Así también, se tiene descrito el tipo de documento de identidad proporcionado para esta alta que corresponde en este caso al código 1, es decir el DNI. Por último, se especifica el tipo de lugar donde reside que para este ejemplo se refiere a una vivienda.

En general, los datos del extranjero se describen de la misma forma que para un habitante nacido en España, sin embargo, no se describen ni el municipio ni la provincia sino el país de su nacimiento mediante la propiedad [espad:paisNacimiento](espad:#paisNacimiento). A continuación, se especifica su nacionalidad mediante la propiedad [schema:nationality](http://schema.org/nationality) enlazada con la correspondiente instancia del país. Luego, se registra el documento de identidad empleado para el registro que es un Pasaporte representado por el código 2 de la respectiva lista de conceptos. Otro detalle que se puede reflejar en un registro de variación de un habitante es el municipio, provincia o el país ya sea de procedencia o de destino del habitante según sea el tipo de variación que se está registrando. Esta particularidad se puede registrar por ejemplo para el caso de un Alta por Cambio de Residencia (ACR) mediante las propiedades [espad:provinciaProcedencia](#provinciaProcedencia), [espad:municipioProcedencia](#municipioProcedencia), o [espad:paisProcedencia](#paisProcedencia) de tal forma que se pueda concocer al lugar que se mudó el habitante. Por otra parte, para el caso de una variación debida a una Baja por Cambio de Residencia se puede representar mediante las propiedades [espad:provinciaDestino](#provinciaDestino), [espad:municipioDestino](#municipioDestino), o [espad:paisDestino](#paisDestino) y así se puede registrar a qué lugar se fue el habitante. En el caso específico del código de nuestro ejemplo se observa que el municipio y provincia de procedencia de este extranjero es el municipio y la provincia de Pontevedra.
